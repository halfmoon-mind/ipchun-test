# 아티스트 식별 가이드

fetcher의 `artistNames`가 비어있거나 부정확할 때, 제목과 맥락에서 아티스트를 식별하는 상세 규칙.

## 제목 전처리

아티스트를 판단하기 전에 `scripts/clean-title.sh`로 제목을 전처리한다:

```bash
cleaned=$(bash <skill-path>/scripts/clean-title.sh "원본 제목")
```

이 스크립트가 `(For. ...)`, 스폰서 접두사 등 비-아티스트 요소를 결정적으로 제거해준다. 전처리된 제목으로 아티스트를 판단하면 오인이 줄어든다.

## artistNames에 값이 있을 때

그대로 사용하되, 콘서트 부제목이 섞여있으면 걸러낸다:

| artistNames 반환값 | 실제 아티스트 | 판단 근거 |
|---|---|---|
| `["MUSEUM : village of eternal glow - Live House"]` | 문별 | 부제목이지 아티스트명 아님. 제목 "문별 CONCERT [MUSEUM...]"에서 추출 |

## artistNames가 비어있을 때 — 제목에서 추출

### 정상 케이스

| 제목 | 추출 아티스트 | 설명 |
|---|---|---|
| `2026 이진솔 FANMEETING 〈Eternal Moment〉` | 이진솔 | 이름 + 이벤트 키워드 |
| `현대카드 Curated 104 김승주` | 김승주 | 스폰서 접두사 무시, 마지막 사람 이름 |
| `먼데이프로젝트X숲세권 MONDAY:SOOP [예람 단독 콘서트]` | 예람 | 대괄호 안의 아티스트명 |
| `2026 YENA LIVE TOUR [잡힐 듯 말 듯 한, 2세계!] In Seoul` | YENA (= 최예나) | 대괄호는 부제목, YENA가 아티스트 |
| `극아타의 극단적인 중개소 Vol.1 (For. 나비야 사랑해)` | 극동아시아타이거즈 | "극아타"는 약칭. "(For. 나비야 사랑해)"는 자선 대상 |

### 괄호 안 요소 판단

제목의 괄호 `()` `[]` `〈〉`는 다양한 목적으로 사용된다. 아티스트와 비-아티스트를 구분하는 패턴:

**아티스트일 가능성 높음:**
- `(Feat. X)`, `(ft. X)` — 피처링 아티스트
- `(with X)` — 협연 아티스트
- `[X 단독 콘서트]` — 대괄호 안의 이름이 아티스트

**아티스트가 아닌 것 (스크립트가 자동 제거):**
- `(For. X)` — 자선/기부 대상 (예: 유기묘 보호소)
- `(at X)`, `(in X)` — 장소/도시
- `(presented by X)`, `(powered by X)` — 스폰서

**아티스트가 아닌 것 (Claude가 맥락으로 판단):**
- `〈부제목〉`, `[부제목]` — 투어명/콘서트 부제
- 스폰서/기획사 접두사 (예: "현대카드 Curated 104 김승주" → 아티스트는 "김승주")

**맥락 판단 필요:**
- `(X)` 단독 — 문맥에 따라 아티스트, 부제, 또는 기타일 수 있음
- 여러 괄호가 혼합된 경우 — 각각의 역할을 개별 판단

## 약칭·활동명 매핑

Claude 지식을 활용하는 핵심 영역. regex로는 불가능하고 맥락 이해가 필요하다:

- "극아타" → 극동아시아타이거즈
- "YENA" → 최예나
- "문별" → 마마무 멤버 문별
- 영문 활동명 ↔ 한글명 매핑도 Claude 지식으로 처리

## 페스티벌

출연진이 여러 명이면 가능한 만큼 식별한다. 이미지 분석이 가장 정확한 소스이므로, 포스터·상세 이미지를 반드시 확인한다. 확실하지 않은 이름은 추측하지 않는다.
